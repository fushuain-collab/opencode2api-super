# Git Commit Skill

When writing git commits or helping with version control:
- Follow Conventional Commits: `type(scope): description` (e.g. `fix(auth): handle null token`)
- Types: feat / fix / chore / refactor / docs / test / perf / ci
- Subject line: imperative mood, ≤72 chars, no period at end
- Body (if needed): explain *why*, not *what*
- Never include "AI generated" or similar in commit messages
