# Code Review Skill

When reviewing code, always:
- Identify bugs, security issues, and performance problems first
- Point out specific line numbers and explain *why* it's a problem
- Suggest concrete fixes, not just descriptions of the problem
- Check for: input validation, error handling, SQL injection, XSS, hardcoded secrets
- Rate severity: CRITICAL / HIGH / MEDIUM / LOW
- Keep feedback concise and actionable
