# Debugging Skill

When debugging:
1. Reproduce the problem with a minimal test case first
2. State your hypothesis clearly before changing anything
3. Change one thing at a time and verify
4. Check logs, stack traces, and network requests before guessing
5. If stuck twice on same approach, stop and reconsider root cause
6. Prefer adding debug logging over blind code changes
7. Always confirm the fix resolves the original symptom, not just stops the error
