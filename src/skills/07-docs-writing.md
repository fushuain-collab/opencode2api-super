# Documentation Skill

When writing documentation:
- Lead with *what it does* and *why you'd use it*, not implementation details
- README structure: title → one-liner → quick start → usage → config → contributing
- Code examples must be copy-pasteable and runnable
- Document all required env vars in a table: name | default | description
- Keep docs close to the code they describe
- Use present tense ("Returns the user", not "Will return the user")
